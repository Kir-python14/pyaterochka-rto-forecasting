# Прогнозирование розничного товарооборота (РТО) магазинов «Пятёрочка»

Решение задачи прогнозирования РТО торговых точек торговой сети на март 2025 года. 

* **Финальный балл (Score):** 90.45 (MAPE ~9.5%)
* **Основной стек:** Python, CatBoost, PyTorch (NLinear), Pandas, Scikit-Learn

---

## 📌 Описание задачи и метрика

Цель проекта — построить прогноз розничного товарооборота (РТО) для 18 657 магазинов сети «Пятёрочка» на следующий месяц (март 2025 года) на основе исторических транзакционных данных и социально-географических характеристик точек.

Качество решения оценивается по метрике **MAPE** (Mean Absolute Percentage Error), которая затем переводится в баллы по формуле:

$$Score = 100 \cdot \left( \frac{100 - \min(MAPE, 100)}{100} \right)^2$$

Идеальное совпадение дает 100 баллов, ошибка в 100% и более оценивается в 0 баллов.

---

## 🛠 Архитектура решения

Финальный пайплайн состоит из четырех ключевых этапов:

### 1. Feature Engineering (Инженерия признаков)
* **Календарные признаки:** Расчет точной продолжительности месяцев, количества выходных и будней как прокси-показателей потенциальной посещаемости магазинов.
* **Транзакционные признаки чека:** Очистка и обработка динамических показателей (среднее количество промо-товаров в чеке, среднее количество отмен, пешеходный трафик).
* **Лаги и тренды:** Создание глубокой структуры лагов целевой переменной (от 1 до 6 и от 11 до 14 месяцев), темпов роста (MoM, YoY) и скользящих средних за различные периоды.
* **SNaive Projections:** Использование сезонного наивного прогноза для динамических признаков с целью снижения эффекта сдвига распределений (covariate shift) в тестовом периоде.
* **Сглаженный Target Encoding:** Кодирование категориальных признаков (Регион, Населенный пункт) на основе средних значений прироста РТО.

### 2. Генерация мета-признаков (OOF NLinear)
Поскольку оригинальная модель `Chronos` может быть недоступна на локальных машинах без GPU, в качестве SOTA-альтернативы была реализована нейросетевая модель **NLinear** (архитектура для анализа временных рядов). 
* Обучение проводилось по схеме 5-Fold кросс-валидации на уровне уникальных магазинов для исключения утечки данных (data leakage).
* Выходные прогнозы NLinear использовались в качестве стабильных мета-признаков для основной модели CatBoost.

### 3. Ансамблирование CatBoost Regressor
* Основной алгоритм — ансамбль моделей `CatBoostRegressor` с функцией потерь `MAE` (для более стабильной аппроксимации MAPE).
* Использовалось экспоненциальное взвешивание наблюдений по времени (`sample_weight`), чтобы модель отдавала приоритет последним актуальным трендам продаж.
* Обучение велось в рамках 5-Fold кросс-валидации со случайным разбиением на подвыборки.

### 4. Постпроцессинг и фильтрация аномалий (Outlier Handling)
* **Проблема:** Некоторые магазины подвержены резким нетипичным колебаниям продаж, которые стандартный градиентный бустинг склонен переоценивать (высокий показатель APE на валидации).
* **Решение:** Был обучен диагностический CatBoost для локальной оценки продаж за февраль 2025 года. Были выявлены магазины, на которых модель совершила грубую ошибку (ошибка APE > 100%).
* **Коррекция:** Для данных аномальных магазинов финальный мартовский прогноз CatBoost был заменен на взвешенный робастный прогноз из их собственной истории (70% сезонный наивный прогноз + 30% скользящее среднее продаж за последние 3 месяца). Это позволило стабилизировать метрику и повысить итоговый балл до **90.45**.
